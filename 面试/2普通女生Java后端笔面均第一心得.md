# 普通女生Java后端笔面均第一我的心得（零基础版）

个人情况：

学习时长：4个月

入职岗位：Java后端开发（12K×13薪）

我去年11月开始系统学习Java，普通二本计算机相关专业，最终拿下5个offer。

我自认为资质很普通，大学成绩中等，代码量也不大。这里分享下我作为转行小白的实战经验（重点是日常学习规划），希望能给同在路上的人一点参考。

总结了下自己的学习强度：

1、朋友圈关闭，技术社区全开，看源码、看实战帖

2、每天**4小时雷打不动的编码时间**，不追求单次时长，但保证连续性和专注度

3、上午学习算法+八股文，边吃饭边记1个面试高频考点

4、下午集中实战，杜绝空看理论，每学2小时SpringBoot或Redis，必跟敲1个小项目

5、晚上复盘，用笔记和思维导图固化知识

6、周末沉浸式项目开发，仿开源项目，吃透1个技术点

7.项目驱动，每周至少完成1个完整功能模块不追求大而全，但要闭环。例如学完Spring Boot后，用3天实现“用户管理模块”，从Controller层到数据库表独立设计，连调Postman测通所有接口，卡住就查官方文档或Debug跟流程，吃透数据流转逻辑。

8.错题当日消化，分类记录：①框架配置类错误、②源码理解偏差、③业务逻辑漏洞。每页标注错误场景、解决方案、底层原理，周末集中复盘。

9.别迷信天赋，代码量是唯一硬标准。

10.高效学习=源码+实践+总结，听框架课容易陷入“懂了的幻觉”，必须结合动手。

11.核心框架至少学两遍，拒绝浅尝止。第一遍快速实现功能，第二遍专攻“为什么”。第二遍学习时，边读源码注释边画时序图，磨刀不误砍柴工。

12.面试冲刺阶段主攻“项目深度”，用STAR法则重构项目描述。

13.刷GitHub时发现优秀代码，立刻摘录其设计思路和精准术语，模仿用到自己项目中。技术表达专业化，能极大提升面试官好感度。

14.断绝后路，用“生存倒逼成长”。我注销所有游戏账号，手机只留钉钉、GitHub、掘金。提前3个月投简历，哪怕被拒也逼自己直面差距。转行没有“试试看”，只有“必须赢”。



第一阶段：现在-12.30

学习模板：以“理解概念→动手实践→构建小项目”为核心循环。

学习重点：

1.数据库：学习MySQL的基本语法（增删改查）、事务特性（ACID）和索引的基本原理。

2.Java Web基础：了解Servlet的生命周期和作用，理解HTTP协议的基本知识，并学会使用Maven进行依赖管理。此阶段不必深究过时的技术，如JSP，重点是理解Web请求-响应模型。

3.SSM框架：按顺序学习Spring、Spring MVC和MyBatis。理解每个框架解决了什么问题，并尝试进行SSM整合。



第二阶段：1.4-2.15

学习模板：采用“项目驱动学习”的模式。选择一个成熟的项目，亲手敲一遍代码，理解业务逻辑和技术选型。

学习重点：

1.Spring Boot：深入学习并应用Spring Boot，体会其“约定优于配置”带来的便捷性。

2.项目开发：在项目中，要重点掌握分层架构的设计思想，并学习全局异常处理、日志打印和单元测试的实践。

3.Redis：学习其基本数据结构，并尝试在项目中用它实现缓存功能，以提升查询性能。消息队列：了解RabbitMQ或Kafka的基本概念，并在项目中实现异步解耦的场景，比如发送通知或处理耗时任务。



第三阶段：2.16-3.25

学习模板：“广度优先，重点深入”。白天学习新技术概念，晚上复习八股文和刷算法题。

学习重点：

1.微服务与分布式入门：了解Spring Cloud的基本组件。理解分布式中的基本概念，如CAP定理、分布式事务和分布式锁的实现思路。

2.八股文：系统复习Java核心、数据库、框架等常考面试题。

3.算法：坚持在LeetCode上刷题，优先完成Hot100和剑指Offer的题目，重点是理解解题思路，而非死记硬背。

4.项目复盘：对你第二阶段完成的项目进行深度复盘，准备如何清晰地介绍项目背景、技术选型、遇到的难点及解决方案。

学习工具：视频课件、50+练手项目、例题解析等
