## 概念

JVM、JDK、JRE三者关系? 一句话讲明白

JDK 包含 JRE，JRE 又包含 JVM；简单说就是**JDK > JRE > JVM**，三者是层层包含的关系，分别对应 Java 开发、运行、执行的不同层面。

实用场景：开发 Java 程序装 JDK，仅运行 Java 程序装 JRE 即可，JVM 是底层核心无需单独安装。



值传递和引用传递的区别? 一句话讲明白

值传递是把变量的**值的副本**传给方法，方法内修改不会影响原变量；

引用传递是把变量的**内存地址（引用）** 传给方法，方法内通过地址修改对象内容会影响原对象（但 Java 中只有值传递，所谓的 “引用传递” 本质是传递引用的值）。

## 数据类型

long和int可以互转吗?一句话讲明白

long 和 int 可以互转，但 int 转 long 是**自动类型转换（向上转型）** 无精度损失，long 转 int 是**强制类型转换（向下转型）** 可能丢失精度或溢出。



为什么用bigDecimal不用double ?一句话讲明白

double 是浮点数，在存储和计算十进制小数（如金额）时会产生二进制精度丢失，而 BigDecimal 基于十进制算术运算，能精准表示和计算小数，避免精度误差。



装箱和拆箱是什么？一句话讲明白

装箱是 Java 自动将**基本数据类型（如 int）** 转换为对应的**包装类对象（如 Integer）** 的过程，拆箱则是反向将包装类对象转换为基本数据类型的过程，本质是编译器自动帮你调用了包装类的`valueOf()`（装箱）或`xxxValue()`（拆箱）方法。



Integer相比int有什么优点？一句话讲明白

Integer 作为 int 的包装类，支持**null 值表示**、可用于泛型 / 集合框架、提供了丰富的数值操作方法，而 int 仅能存储基础整数值且默认值为 0，无法适配需要对象的场景。



那为什么还要保留int 类型?一句话讲明白

Java 保留 int 是因为它作为基本数据类型，**内存占用更小、运算速度更快**，在纯数值计算场景下性能优势显著，而 Integer 作为对象有额外的内存开销和装箱 / 拆箱的性能损耗，二者各司其职。



说一下 integer 的缓存，一句话讲明白

Java 为了优化性能，会默认缓存`-128 ~ 127`范围内的 Integer 对象，使用该范围的数值时（如自动装箱、valueOf 方法）会复用缓存对象，超出范围则创建新对象。

## 面向对象

多态体现在哪几个方面?一句话讲明白

多态主要体现在**方法重载（编译时多态 / 静态多态）** 和**方法重写（运行时多态 / 动态多态）** 两个核心方面，一句话总结就是：**多态体现在同一类中方法名相同但参数列表不同的重载（编译期确定执行哪个方法），以及子类重写父类方法后，父类引用指向子类对象时调用子类重写方法的重写（运行期确定执行哪个方法）这两个层面**。



多态解决了什么问题？

一句话总结多态解决的核心问题：**多态通过统一接口屏蔽不同实现的差异，解决了代码耦合度高、扩展性差的问题，让程序可以基于抽象编程而非具体实现，新增功能时无需修改原有代码**。



面向对象的设计原则你知道有哪些吗，一句话讲明白

一句话总结面向对象设计原则：**面向对象核心有 7 大设计原则，核心思想是高内聚低耦合、对扩展开放对修改关闭，具体包括单一职责、开闭、里氏替换、依赖倒置、接口隔离、迪米特法则、合成复用，前 5 个是核心，后 2 个是补充**。



重载与重写有什么区别?一句话讲明白

一句话总结重载与重写的区别：**重载是同一类中方法名相同但参数列表（个数 / 类型 / 顺序）不同（编译时多态，看参数匹配），重写是子类对父类同名、同参数、同返回值方法的重新实现（运行时多态，看对象实际类型）**。
